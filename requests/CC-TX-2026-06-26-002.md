Summary:
ConCOREdance distinguishes institutional memory from model memory by making each transmission portable across HTML, PDF, and AI-optimized Markdown, with CANON.md serving as the living model-agnostic index.

Body:
During Ada Vance's onboarding to the ConCOREdance Architecture Council, a key archive limitation became visible: modern AI collaborators cannot be expected to reliably locate and synthesize canon from public websites, Google Drive folders, or platform-specific memory systems.

The answer is not to make every AI search harder. The answer is to make canon portable.

Every archived transmission should produce three official artifacts. Archive HTML remains the beautiful human-readable public presentation layer. Archive PDF provides a printable preservation and sharing copy. AI Markdown provides a clean, structured, stable, minimally styled source that any AI collaborator can consume without relying on proprietary memory or platform-specific retrieval behavior.

The Markdown artifact may become one of the most valuable outputs of the Transmission Archive because it allows ChatGPT, Gemini, Claude, Copilot, and future models to orient from the same project-owned source of truth.

The archive should also maintain a living CANON.md index. That index should list the current active canon with each transmission ID, title, status, layer, brief abstract, and relative links to HTML, PDF, and Markdown versions. Future AI collaborators can then be onboarded with CANON.md plus the specific transmissions relevant to the current discussion.

This creates a clear distinction between institutional memory and model memory. Institutional memory belongs to the project. Model memory belongs to an individual AI. Those should never be treated as the same thing.

By generating standardized Markdown canon and a portable index, ConCOREdance becomes model-agnostic. It coordinates intelligence rather than depending on any single AI's memory.

The archive becomes the project's memory.

Decisions:
- Every published transmission should produce Archive HTML, Archive PDF, and AI Markdown artifacts.
- CANON.md should serve as the living canonical index for AI collaborators and human maintainers.
- Institutional memory and model memory are distinct and should remain distinct.
- ConCOREdance should remain model-agnostic by preserving canon in project-owned files.
- AI collaborators should be onboarded from CANON.md and the specific relevant transmissions, not from proprietary assistant memory alone.

Next Actions:

- Update the GitHub canonization workflow so CANON.md is committed with future transmissions.
