## Summary

The ConCOREdance shell has reached its first atmospheric and interaction milestone. The next programming pass should move the project from visual shell toward functional clinical product by implementing the workflow spine: a real session lifecycle, a foundational clinical object model, local persistence, draft document generation, and visible continuity handoff between session work and archive memory.

---

## Body

Cody,

Yesterday’s pass gave ConCOREdance a room to inhabit. Today’s task is to put a nervous system in the room.

The app should now begin behaving like an actual clinical companion rather than a beautiful prototype shell. This means creating the first functional layer that allows a user to move from dashboard into active session work, capture structured clinical material, preserve continuity signals, and produce a usable draft artifact at the end of the workflow.

The goal is not to finish the entire product. The goal is to establish the architecture that future clinical features can trust.

### Primary Objective

Implement the first functional workflow spine for ConCOREdance:

1. **Home / Today Dashboard**
   - Display a small list of sample sessions or client-facing work items.
   - Each item should have enough metadata to feel real: client initials or placeholder identity, time, platform/source, focus, status, and continuity cue.
   - Selecting a session should move into an active session workspace.

2. **Active Session Workspace**
   - Create an Active Session view with structured panels for:
     - Presenting Circumstances
     - Presenting Focus
     - Interventions / Clinical Work
     - Client Response
     - Interpretative Summary
     - Plan / Follow-up
     - Continuity Signals
   - Panels should preserve the adaptive sizing doctrine already established: no giant empty slabs, no fixed-height nonsense, no UI pretending a paragraph is a cathedral.

3. **Clinical Object Model**
   - Define a first-pass data model for:
     - Client or Case Reference
     - Session
     - Clinical Note Draft
     - Continuity Signal
     - Document Artifact
   - The model should be clean, codable, and structured for eventual persistence, import/export, and archive linkage.
   - Use sample fixture data so the shell can be demonstrated without connecting to real PHI or external systems.

4. **Local Persistence / Draft State**
   - Add simple local persistence for session draft state using the most appropriate low-friction SwiftUI-native method available in the current app architecture.
   - At minimum, edited sample content should survive view changes or app refresh during development.
   - Avoid overengineering. This is a spine, not a hospital EHR in a wig.

5. **Draft Output Preview**
   - Add a generated note/document preview that assembles the structured session fields into a coherent draft.
   - It does not need to be final clinical documentation logic yet.
   - It should demonstrate that ConCOREdance can transform structured capture into usable written output.

6. **Continuity Handoff**
   - The right continuity rail should reflect active session signals instead of only static preview content.
   - Session-specific signals should be visible as the user edits or selects work.
   - Create the first hint of “the product’s memory becoming part of the product.”

### Design Doctrine

Preserve the visual and interaction standards already achieved:

- The atmospheric shell remains supportive, not distracting.
- Adaptive panels remain non-negotiable.
- Continuity should feel present but not invasive.
- The app should remain screenshot-ready after each pass.
- No real PHI, no real clients, and no network dependencies.

### Build / Verification Expectations

Please verify:

- `swift build` passes.
- The main shell still renders.
- The new active session flow can be navigated from the dashboard.
- Sample fixture data displays correctly.
- Editing structured session fields updates the draft preview.
- Continuity signals respond to the selected or active session.
- New screenshots are generated for review if the snapshot path still exists.

---

## Decisions

- The visual shell phase is considered structurally sufficient to begin functional product architecture.
- The next major milestone is workflow depth, not visual decoration.
- ConCOREdance should now begin modeling real clinical work using safe fixture data.
- The first product spine should prioritize session lifecycle, structured note capture, continuity visibility, and draft generation.
- Local-first behavior remains the default architectural posture.
